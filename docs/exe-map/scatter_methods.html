This view drops the destructors — the COMDAT-pooled members that MSVC exiles from the TU block (measured dtor median distance-to-sibling 0x7b0, p90 ~645 KB, vs 0xa0 / 93%-within-8KB for methods). What remains is the true per-TU layout. Fragmentation is a property of the retail link layout, not the reconstruction: MSVC/link concatenates each TU's COMDATs in source order, so a TU is mostly one contiguous block. Data grouping is our src/ files (kept close to the original TUs). Source: scatter*.json.
