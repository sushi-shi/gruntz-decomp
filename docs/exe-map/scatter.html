Destructors here still count. MSVC emits the vtable-referenced deleting destructor as a COMDAT and the linker pools those (plus the ~Class bodies) into shared low-address runs away from the TU block — so they inflate the scatter. Fragmentation is a property of the retail link layout, not the reconstruction: MSVC/link concatenates each TU's COMDATs in source order, so a TU is mostly one contiguous block. Data grouping is our src/ files (kept close to the original TUs). Source: scatter*.json.
